# fp-type-dd-experiment


# 大本の発想
モナドを使う利点
- 型定義レベルで、そのドメインがどんな文脈に着目すべきかが明示的になる
- bindチェインにより、文脈内部では文脈を気にしないでいられる
↓
「必要なパイプラインの型シグネチャだけ定義してpushしたら、その中身を実装してくれる」みたいなAIハーネス作ったら、AI任せになりながらかなり安全に実装できるんじゃないか？？？？？
いざ具体を見るときの可読性も、文脈内で文脈を無視できるからかなり読みやすいはず

## 仮説
- 型が強い言語ほどAIへの指示が短くなり、トークン消費が減る
  - 実装速度も上がる
- 型の強さとAIへの指示コストが反比例する
- 型が弱い言語 or 指示が自然言語などで不明瞭
  - 安全性が大きく下がる
  - 生成されたコードの可読性が下がる
